# 1 — Core concept & experience (one-sentence)

A warm, side-scrolling interactive story where she chooses an anime-style avatar, walks through a field of memories, clears a gentle storm, climbs a mountain of affirmations, and taps your silhouette on a sunrise cliff to receive the marriage proposal.

---

# 2 — Emotional tone & design language

* Tone: intimate, tender, cinematic, calming — like a short Studio Ghibli scene.
* Palette: soft pastels with warm sunrise accents.

  * Sky Blue: `#CDE7FF`
  * Pale Peach: `#FFE6D6`
  * Warm Sunset Orange: `#FFB57A`
  * Meadow Green: `#BEE7C6`
  * Petal Pink: `#FFD6E0`
  * Shadow Gray: `#6D6D6D` (for subtle contrast)
* Typography:

  * Heading: rounded friendly serif (e.g., "Merriweather" or "Poppins" bold)
  * Body: clean rounded sans (e.g., "Inter" / "Nunito")
* Lighting: soft bloom, subtle vignette around edges.
* Motion: slow, organic — no jarring animations.

---

# 3 — High-level flow & UX map

1. **Landing / Avatar selection** (`/`)

   * Soft intro, choose avatar, start button.
2. **Field of Memories** (`/memories`)

   * Side-scroll right. Click 5 balloons (memories). Each pops and shows a modal message. After 5, continue.
3. **Storm of Shadows** (`/storm`)

   * Rain & clouds. Tap clouds to dissolve. Each reveals a heartfelt line. After all cleared, continue.
4. **Mountain of Love** (`/mountain`)

   * Upward scroll. Each step reveals affirmations. Reach summit.
5. **Sunrise Cliff** (`/sunrise`)

   * Tap silhouette (you) → world brightens, proposal text fades in, two buttons (YES / OF COURSE YES) → final screen.
6. **Proposal Complete** (`/complete`)

   * Petal shower, heart sunrise, optional auto-play short voice clip (you saying “I love you…”), share/download screenshot option.

# 5 — Pages: micro-behavior, exact copy, animation specs

All timings use seconds. Easing functions: `cubic-bezier(0.22, 0.8, 0.2, 1)` (gentle spring-like).

## AvatarSelection.tsx

**Layout**

* Full-screen center card with watercolor sky background (`bg_meadow.png` blurred).
* Title: `Choose the one who will walk this journey`

  * Font-size: 28px (desktop), 20px (mobile)
  * Centered, soft shadow.
* Avatar cards (3) in a row (wrap on mobile).

  * Each card: circular frame, 140px (desktop) / 96px (mobile), hover scale `1.06`, transform origin center, shadow & glow.
  * On click: animate bounce (scale 0.95 → 1.12 → 1), emit 6 particle petals (use 6 `petal1.png` sprites with random velocity, fade).
* Copy below:

  * `"Every journey starts with a choice. I’m happy you chose to walk mine."`
  * opacity fade-in 0 → 1 at 0.6s, duration 0.8s.
* Start button: `Start the Journey`

  * Primary color: `#FFB57A` (sunset); hover raises 6px; click depress animation (0.04s).
* Accessibility:

  * Each avatar is a button with `aria-pressed` state.
  * Keyboard: left/right arrow to change selection, Enter to select.

**Exact copy strings**

* Title: `Choose the one who will walk this journey`
* Subtext: `Every journey starts with a choice. I’m happy you chose to walk mine.`
* Button: `Start the Journey`

## FieldOfMemories.tsx

**Scene**

* Parallax layers: distant hills, mid grass, foreground grass (3 layers) — horizontal parallax as avatar walks right.
* Auto-walk behaviour: avatar slowly moves right when user taps the screen (or presses → right arrow). On desktop, they can also drag or click to move.
* 6 floating balloons placed along path at these X positions: 1200px, 2400px, 3400px, 4600px, 5800px, 7000px. (Use units relative to container width on responsive).
* Each balloon: bobbing animation `translateY(-6px)` to `translateY(6px)` duration `2s` infinite.
* Balloon click:

  * Play `pop.wav` (60ms delay) then particle burst (10 sparkles).
  * Modal card appears bottom-left with memory text; modal width `min(420px, 90vw)`.
  * Modal copy (five from earlier; for the sixth use: `“I love how you laugh when you think no one’s watching.”`)
  * Modal shows small icon and timestamp (e.g., `First Date — July 14th` — these are example placeholders you can replace).
* Counter: show `Memories collected: X/5` anchored top-left.
* Continue: after any 5 distinct balloons popped, a subtle golden petal path animates leading to a “Continue” floating step (an orb). The Continue orb pulses + glow. Button label: `Continue →` (keyboard accessible).
* Soundscape: soft ambient loop already playing (AudioPlayer toggled on/off).

**Exact memory modal messages**

1. `This day changed everything.`
2. `This moment is still my favorite.`
3. `Your smile still makes me melt.`
4. `You made even the ordinary feel magical.`

## StormOfShadows.tsx

**Scene**

* Background darkens with a bluish-gray overlay (`rgba(25,35,52,0.5)`).

* Rain layer: thin vertical streaks with randomized start times; parallaxed slower than foreground.

* Clouds: 4 cloud sprites drift down left-to-right; each cloud contains shadow text and a small icon (paper, tiny clock, greyed heart, knot).

* Tap cloud:

  * Play `cloud_poof.wav` and an animated swirl dissolving the cloud (mask reveal).
  * A warm ray of light replaces the cloud position.
  * The revealed text appears center-screen for 1.6s in a soft opaque bubble with slightly warmer background.

* Cloud messages (order randomized each run):

  1. `You helped me through my fears.`
  2. `You stayed, even when I pushed you away.`
  3. `You were my calm during every storm.`
  4. `You made me feel safe again.`

* After clearing all 4 clouds: sun rays animate from center with a bloom effect (duration 1.4s), rainbow arc appears; small `Move Forward` button displayed with text: `Because of you, even storms felt survivable. →`

**Animations & timings**

* Cloud dissolve: mask-scale animation 0 → 1.2 over 0.6s with `easeOut`.
* Rain intensity reduces as each cloud cleared, and ambient soundtrack brightens in volume (fade +6dB each cloud).

**Accessibility**

* Clouds focusable via Tab, `Enter` to clear, `aria-label` with message text.

# 6 — Component contract (props, events)

### AvatarCard.tsx

* Props: `{ id: string; name: string; imgSrc: string; alt: string; selected: boolean; onSelect(id) }`
* Emits: `onSelect(id)`; toggles `aria-pressed`.

### BalloonMemory.tsx

* Props: `{ id: string; x: number; y: number; message: string; popped: boolean; onPop(id) }`
* Emits: `onPop(id)` when clicked.

### CloudTap.tsx

* Props: `{ id: string; message: string; onClear(id); cleared }`
* Emits: `onClear(id)`; internally plays dissolve animation.

### StepAffirmation.tsx

* Props: `{ id: number; text: string; active: boolean }`
* Visual: tiny glow bubble when `active`.

### ParallaxBackground.tsx

* Props: `{ layers: Array<{src, speed}>; offset }`
* Handles transform for parallax using requestAnimationFrame or Framer Motion.

### AudioPlayer.tsx

* Props: `{ trackSrc, loop?, volume?, onToggle }`
* UI: small sound toggle pinned bottom-right. Remember: start muted by default on mobile to comply with autoplay policies; play after first user interaction.

# 8 — Accessibility & keyboard navigation (detailed)

* All interactive elements are `button` elements or have role `button`.
* Manage focus order logically: Avatar → Start Button → In-game controls.
* Add `Skip to content` link top-left; `AccessibilitySkip.tsx`.
* Aria-live region for dynamic proposal text so screen readers announce lines when they appear:

  * `<div aria-live="polite" aria-atomic="true">{currentProposalLine}</div>`
* Ensure color contrast: text on main interactive buttons must have contrast ratio >= 4.5:1.
* Allow keyboard controls: Arrow keys navigate & interact (left/right/up/down), Enter/Space to activate.

# 11 — Performance & production tips

* Use `will-change: transform` only during animations and remove after.
* Use `image-set` for responsive images (`1x`, `2x`) — or `srcset`.
* Lazy-load distant background layers with `loading="lazy"` where supported.
* Keep sprite counts small: particle bursts should spawn max 60 and be removed after animation.
* Use CSS transforms for GPU acceleration; avoid layout thrashing.
* Tree-shake unused Framer Motion features.

2. `cd journey-through-us`
3. `npm install`
4. Add Tailwind: follow Tailwind + Vite official setup.
5. Add Framer Motion: `npm install framer-motion`
6. Copy assets into `public/` as above.
7. Start dev: `npm run dev`
8. Build: `npm run build`
9. Deploy:
